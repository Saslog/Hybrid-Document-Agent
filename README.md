# Hybrid-Document-Agent
Hybrid Document Agent: Intelligent Query Routing for Cost-Optimized RAG Systems

Abstract

Large language models (LLMs) deployed in enterprise document intelligence systems face a fundamental cost-latency tradeoff: powerful cloud-based LLMs deliver superior reasoning capability but incur prohibitive per-query inference costs at scale, while lightweight edge-deployed models reduce cost by two orders of magnitude but lack the multi-step reasoning capacity required for complex analytical queries. This paper presents the Hybrid Edge-Cloud Document Agent (HECA), a production-grade retrieval-augmented generation (RAG) system that resolves this tradeoff through intelligent, query-level routing. HECA employs a five-component pipeline: (1) a lightweight NLP query classifier that computes complexity, sensitivity, and multi-document scores without invoking any heavyweight model; (2) a cost-aware decision-tree router implementing six priority-ordered routing branches; (3) a semantic cache layer using cosine-similarity lookup to serve repeated queries at zero inference cost; (4) a hybrid retrieval pipeline combining BM25 sparse retrieval with dense bi-encoder vector search, fused via Reciprocal Rank Fusion (RRF) and re-ranked by a cross-encoder; and (5) inference adapters for three execution tiers: edge (Mistral-7B Q4, Phi-3-Mini), hybrid (edge retrieval with cloud synthesis), and cloud (GPT-4o, Claude-3). Experimental evaluation on a diverse enterprise query benchmark demonstrates that HECA achieves 72–85% cost reduction versus cloud-only baselines with less than 2.4% accuracy degradation on routable query classes. The full implementation, comprising 1,400 lines of dependency-free Python across 15 modules, is made publicly available.
